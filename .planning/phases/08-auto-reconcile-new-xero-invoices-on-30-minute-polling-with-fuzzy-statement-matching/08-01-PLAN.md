Create the backend polling infrastructure that fetches new Xero invoices every 30 minutes, saves them to MongoDB, and fuzzy-matches them against uploaded bank statement records.

Purpose: Automate Xero invoice reconciliation so users no longer need to manually trigger invoice fetches or re-upload statements to get matches.

Output: Two new Mongoose models (XeroSyncState, ReconLog), a background polling service (xeroPollingService.js), and server.js wired to start the poller after DB connect. node-cron installed.

Create `services/xeroPollingService.js` implementing the full polling loop. This is the core of the phase.

**Structure overview:**
```
exports.start() → cron.schedule('*/30 * * * *', runPollCycle)
runPollCycle() → lock check → pollAndReconcile() → unlock
pollAndReconcile() → getValidXeroToken() → fetchNewXeroInvoices() → upsertInvoices() → matchAndReconcile() → writeLog()
```

**Detailed implementation instructions:**

1. **Imports required:**
   - `node-cron` for scheduling
   - `axios` for Xero API calls (already installed)
   - `{ AuthorizationCode }` from `simple-oauth2` (already installed)
   - `XeroTenants` from `../modals/xeroTenantsModal`
   - `XeroSyncState` from `../modals/xeroSyncStateModal`
   - `ReconLog` from `../modals/reconLogModal`
   - `Invoice` from `../2.0/modals/invoiceModal`
   - `{ nameSimilarity }` from `../2.0/scripts/scripts.js`

2. **`getValidXeroToken()` function:**
   - Copy the token refresh pattern from `AuthController.js` (the `optionalXeroTokenInfo` middleware).
   - Create an `AuthorizationCode` client with `process.env.XERO_CLIENT_ID` and `process.env.XERO_CLIENT_SECRET`.
   - Token host: `https://identity.xero.com/connect/token`, auth path: `https://login.xero.com/identity/connect/authorize`, token path: `/connect/token`.
   - Read `XeroTenants.findOne().lean()`. If no tenant or no `authData.refreshToken`, return null (Xero not connected — skip cycle silently).
   - Check if token is expired. If expired, refresh it and write the new access token, refresh token, and expiry time back to XeroTenants.
   - Return `{ accessToken, tenantId }`.

3. **`fetchNewXeroInvoices(accessToken, tenantId, lastPolledAt)` function:**
   - Build headers: `Authorization: Bearer ${accessToken}`, `Xero-tenant-id: tenantId`, `Accept: application/json`.
   - If `lastPolledAt` is not null, add `If-Modified-Since` header with the ISO string **without milliseconds**: `new Date(lastPolledAt).toISOString().replace(/\.\d{3}Z$/, 'Z')`.
   - Paginate: start at page 1, page size 100. Loop until the response returns fewer than 100 invoices.
   - URL: `https://api.xero.com/api.xro/2.0/Invoices?page=${page}&pageSize=100`
   - Add 300ms delay between pages to avoid rate limits: `await new Promise(r => setTimeout(r, 300))`.
   - Collect all invoices from all pages into a single array and return it.

4. **`getIdScore(fileInv, xeroInv)` function — replicate from upload:**
   - Define `idNoLetters = (s) => String(s ?? "").replace(/[a-zA-Z]/g, "")`.
   - Read `xeroInv.invoiceNumber`. If empty, return 0.
   - Get `potentialIds` from fileInv: use `fileInv.potentialInvoiceIds` if non-empty array, else fallback to `[fileInv.invoiceNumber, fileInv.referenceId, fileInv.id].filter(Boolean)`.
   - For each potential ID, compute `nameSimilarity(digits(pid) || pid, digits(xeroNum) || xeroNum)`. Track best score.
   - Return `Math.round(best * 1e6) / 1e6`.
   - IMPORTANT: This function is asymmetric. The first argument (fileInv) is the statement record, the second (xeroInv) is the Xero invoice. This is correct per research — the statement record has `potentialInvoiceIds`.

5. **`matchAndReconcile(xeroInvoices)` function:**
   - Define `MATCH_THRESHOLD = 0.8` (same as upload).
   - For each Xero invoice in the batch:
     a. **Upsert to MongoDB:** `Invoice.findOneAndUpdate({ invoiceNumber: xeroInv.InvoiceNumber, fromXero: true, contactId: xeroInv.Contact?.ContactID }, { $set: { ... mapped fields ... } }, { upsert: true, new: true })`. Map fields: `invoiceNumber` = `InvoiceNumber`, `amount` = `SubTotal || Total || null`, `status` = `Status === 'PAID' ? 'paid' : 'unpaid'`, `currency` = `CurrencyCode || null`, `date` = `DateString ? new Date(DateString) : null`, `dueDate` = `DueDateString ? new Date(DueDateString) : null`, `contactId` = `Contact?.ContactID || null`, `description` = `LineItems?.[0]?.Description || null`, `fromXero` = `true`, `isDeleted` = `false`, `modifiedLast` = `new Date()`.
     b. **Find statement records to match against:** `Invoice.find({ contactId: xeroInv.Contact?.ContactID, fromXero: false, isDeleted: { $ne: true } }).lean()`. If no contactId on the Xero invoice, skip matching (but still upsert).
     c. **Score each statement record:** call `getIdScore(statementRecord, { invoiceNumber: xeroInv.InvoiceNumber })`. Track the best match and score.
     d. **If bestScore >= MATCH_THRESHOLD:** update the statement record: `Invoice.findByIdAndUpdate(bestMatch._id, { $set: { invoiceNumber: xeroInv.InvoiceNumber } })`. This overwrites any existing invoice number per the locked decision.
   - Return counts: `{ invoicesSaved, matchesWritten }`.

6. **`pollAndReconcile()` function:**
   - Record start time: `const startTime = Date.now()`.
   - Call `getValidXeroToken()`. If null, log `[XeroPoller] Xero not connected — skipping cycle` and return (don't create a log entry).
   - Read `XeroSyncState.findOne().lean()` to get `lastPolledAt`.
   - Call `fetchNewXeroInvoices(accessToken, tenantId, lastPolledAt)`.
   - If no new invoices, update `XeroSyncState` (upsert) with `lastPolledAt: new Date()` and `lastSuccessAt: new Date()`, create a ReconLog with 0 counts, and return.
   - Call `matchAndReconcile(newInvoices)`.
   - Update `XeroSyncState` with `lastPolledAt: new Date()` and `lastSuccessAt: new Date()`.
   - Create `ReconLog` document: `{ ranAt: startTime, durationMs: Date.now() - startTime, newInvoicesFetched: invoices.length, invoicesSaved, matchesWritten }`.
   - Log summary: `[XeroPoller] Cycle complete: ${invoices.length} fetched, ${invoicesSaved} saved, ${matchesWritten} matched (${durationMs}ms)`.

7. **`runPollCycle()` function with lock:**
   - Module-level `let isPolling = false;`.
   - If `isPolling`, log skip message and return.
   - Set `isPolling = true`, wrap `pollAndReconcile()` in try/catch/finally.
   - On error: log error message to console, create ReconLog with error field set.
   - Finally: set `isPolling = false`.

8. **`start()` function:**
   - `cron.schedule('*/30 * * * *', runPollCycle)`.
   - Log: `[XeroPoller] Started — polling Xero every 30 minutes`.
   - Also run one immediate cycle: `runPollCycle()` (fire-and-forget — do NOT await).

9. **Export:** `module.exports = { start }`.

**Modify `server.js`:**
- Add `const xeroPollingService = require('./services/xeroPollingService');` at the top with other requires.
- Inside the IIFE, after `console.log("MongoDB connected successfully");` and before the `catch` block, add: `xeroPollingService.start();`.
- This ensures the poller starts only after MongoDB is connected.
