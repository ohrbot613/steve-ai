<objective>
Fix the mismatch between the Xero Bank Summary report's "Statement Balance" and "Balance in Xero" columns as displayed on the dashboard. The backend silently falls back to statement balance when the Xero balance column is not found in the report, causing the dashboard to show potentially incorrect values with no indication of the source. This plan fixes the backend parsing, cleans up debug artifacts, and updates the frontend to clearly display which balance is shown and flag when the two values differ.

Purpose: Users need accurate bank balance information on the dashboard. When the bank statement balance differs from Xero's computed balance (common due to authorized-but-uncleared transactions), the user must be aware of the discrepancy rather than seeing a single unlabeled number.

Output: Backend returns clean, correctly-parsed balance data with explicit nulls instead of silent fallbacks. Frontend displays the balance with a source indicator and shows the statement balance underneath when it differs from the Xero balance.
</objective>

**Part A: Update `useXeroBalance` hook** (`PDF automation/client/src/hooks/useXeroBalance.js`):

Enhance the hook to compute and return both balance totals and a mismatch flag. In the `.reduce()` block (around lines 47-58), change the logic to compute two separate totals:

```javascript
const filtered = data.accounts
    .filter(acc => acc.status === 'ACTIVE' && acc.currencyCode === data.baseCurrency);

const xeroTotal = filtered.reduce((sum, acc) => {
    const bal = acc.xeroBalance != null ? acc.xeroBalance : acc.statementBalance;
    return sum + (bal || 0);
}, 0);

const statementTotal = filtered.reduce((sum, acc) => {
    return sum + (acc.statementBalance || 0);
}, 0);

// Determine which source is being used
const hasXeroBalance = filtered.some(acc => acc.xeroBalance != null);
const hasMismatch = hasXeroBalance && Math.abs(xeroTotal - statementTotal) > 0.01;
```

Update the `setBalance` call to include the new fields:
```javascript
setBalance({
    total: xeroTotal,          // Primary display (unchanged behavior)
    statementTotal,            // For mismatch indicator
    currency: data.baseCurrency,
    accounts: data.accounts,
    source: hasXeroBalance ? 'xero' : 'statement',
    hasMismatch
});
```

The `total` field remains the same (prefers xeroBalance) so existing consumers are not broken. The new `statementTotal`, `source`, and `hasMismatch` fields are additive.
