## Decisions Made

1. **threadId as String UUID v4 (not ObjectId):** Chosen for external identifier consistency. UUIDs are generated by application layer and passed to both Thread and Message models, avoiding ObjectId reference mismatches.

2. **Message.threadId as String (not ObjectId ref):** Must match Thread.threadId type. Since Thread.threadId is String UUID, Message.threadId is also String for consistent linking.

3. **metadata as Mixed type:** Provides flexibility for storing tool call objects and usage statistics without rigid schema constraints. Sub-fields (toolCalls array, usage object) defined as optional nested Mixed types.

4. **Inline index removal for single fields:** Removed redundant schema.index() calls for threadId and userId that were already declared inline with `index: true`, keeping only the compound indexes.
