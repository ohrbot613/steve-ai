**1. Create `PDF automation/client/src/hooks/useXeroBalance.js`:**

Create a custom React hook that fetches the bank balance from the existing backend endpoint. Follow the codebase convention of using `fetch` (NOT axios -- the codebase uses native fetch for all API calls, see Home.jsx, Login.jsx, AskSteve.jsx, etc.).

The hook should:
- Use `useState` for `balance` (null initially), `loading` (true initially), `error` (null initially)
- Use `useEffect` with empty dependency array to fetch once on mount
- Call `GET /api/v2/scripts/get-bank-balance` with `credentials: 'include'` (JWT cookie auth -- see Top.jsx handleLogout for pattern)
- On success (`response.data.success === true`): Sum all ACTIVE accounts where `currencyCode` matches `baseCurrency`, using `xeroBalance` (preferred) with fallback to `statementBalance`. Set balance to `{ total, currency: baseCurrency, accounts }`.
- On 401 response: Set balance to null (not an error -- user not connected to Xero). Do NOT show error state.
- On other errors: Set error to `'Could not load bank balance'` (sanitized, no raw API errors)
- Always set loading to false in finally block
- Return `{ balance, loading, error }`

**Important response shape from backend** (from `scripts.js` lines 457-523):
```json
{
  "success": true,
  "baseCurrency": "GBP",
  "accounts": [
    {
      "accountId": "abc-123",
      "name": "Business Current Account",
      "currencyCode": "GBP",
      "baseCurrency": "GBP",
      "statementBalance": 23340.00,
      "xeroBalance": 23450.50,
      "status": "ACTIVE"
    }
  ]
}
```
