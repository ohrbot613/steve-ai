    **Part A: Create the useStreamingChat hook**

    Create NEW file `PDF automation/client/src/hooks/useStreamingChat.js` using ES module format (frontend uses ESM).

    The hook exports: `export function useStreamingChat(endpoint)`

    **State:**
    - `messages` (array) -- conversation messages: `{ id, role, content, streaming, toolStatus }`
    - `isStreaming` (boolean) -- whether a stream is active
    - `threadId` (string|null) -- current thread ID, persisted to sessionStorage
    - `error` (string|null) -- last error message
    - `activeTool` (string|null) -- name of tool currently being used (from tool_start events)

    **On mount:**
    - Load `threadId` from `sessionStorage.getItem('askSteve_threadId')` if available

    **sendMessage(content, context) function:**
    1. Set `isStreaming = true`, clear error
    2. Create unique assistant message ID: `const assistantMsgId = 'assistant-' + Date.now() + '-' + Math.random()`
    3. Add user message and empty assistant placeholder to messages:
       ```javascript
       setMessages(prev => [...prev,
         { id: 'user-' + Date.now(), role: 'user', content },
         { id: assistantMsgId, role: 'assistant', content: '', streaming: true }
       ]);
       ```
    4. Build SSE URL:
       ```javascript
       const url = new URL(endpoint, window.location.origin);
       url.searchParams.set('message', content);
       if (threadId) url.searchParams.set('threadId', threadId);
       if (context) url.searchParams.set('context', JSON.stringify(context));
       ```
    5. Create EventSource: `const eventSource = new EventSource(url.toString(), { withCredentials: true })`
       - IMPORTANT: `withCredentials: true` sends cookies for authentication. This works because the backend uses cookie-based JWT auth.
       - Store in ref: `eventSourceRef.current = eventSource`

    6. **Event listeners:**
       - `token` event: Parse `event.data` as JSON, extract `token`. Update the assistant message by appending the token to content:
         ```javascript
         setMessages(prev => prev.map(msg =>
           msg.id === assistantMsgId
             ? { ...msg, content: msg.content + token }
             : msg
         ));
         ```
       - `tool_start` event: Parse data, extract `tool` name. Set `activeTool` state. Optionally update assistant message toolStatus.
       - `tool_end` event: Clear `activeTool` state.
       - `end` event: Parse data, extract `threadId`. Update state:
         - `setThreadId(newThreadId)` and persist: `sessionStorage.setItem('askSteve_threadId', newThreadId)`
         - Mark assistant message as not streaming: `{ ...msg, streaming: false }`
         - `setIsStreaming(false)`
         - `eventSource.close()`
       - `error` event (EventSource onerror): Set error state, `setIsStreaming(false)`, `eventSource.close()`

    7. **Cleanup on unmount** via useEffect return: close eventSourceRef if open.

    **clearThread() function:**
    - Reset messages to empty
    - Set threadId to null
    - Remove from sessionStorage: `sessionStorage.removeItem('askSteve_threadId')`
    - Close any active EventSource

    **Return:** `{ messages, isStreaming, error, activeTool, threadId, sendMessage, clearThread }`

    4. **Remove localStorage persistence code** -- messages are now server-persisted via MongoDB through the streaming service. Remove:

    5. **Update handleSubmit:**

       ```javascript
         e?.preventDefault();
         const messageText = inputValue.trim();
         if (!messageText || isStreaming) return;
         setInputValue('');

         sendMessage(messageText, context);
       };
       ```
       - Remove the existing fetch call, addMessage calls, response parsing, error handling
       - The hook handles all of that internally
       - Replace `isSubmitting` checks with `isStreaming` from the hook

    6. **Update handleReset:**
       ```javascript
       const handleReset = () => {
         clearThread();
         setInputValue('');
         setShowUploadWidget(false);
         setTableData(null);
       };
       ```
       - Remove all the localStorage/sessionStorage/Cedar clearing code
       - The hook's clearThread() handles state cleanup

    7. **Remove `isSubmitting` state** -- replaced by `isStreaming` from hook.

    8. **Update typing indicator**: Change condition from `isSubmitting` to `isStreaming` in the JSX.

    9. **Add tool status display**: When `activeTool` is not null and `isStreaming` is true, show a small status indicator above or within the typing indicator:

       ```jsx
             </div>
           </div>
         </motion.div>
       )}
       ```

    10. **The messages.map rendering stays the same** -- the hook provides messages in the same `{ id, role, content }` shape. The only addition is that streaming messages have `streaming: true` which can optionally show a cursor animation.

    11. **For streaming messages**, add a blinking cursor after the content:


For now, remove the `showUploadWidget` and `tableData` conditional renders, OR keep them but they won't be triggered during streaming. The streaming endpoint focuses on text token delivery. Upload intent and table data features can be added to the streaming protocol in a future iteration if needed.
